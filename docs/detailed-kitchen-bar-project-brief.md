# Kitchen Bar Project Brief

## Project Overview

I want to replace an existing metal railing in my split-level home with a bar that includes cabinet storage. The kitchen sits higher than the adjacent living room, creating a unique opportunity for a multi-functional bar/counter space.

This is sometimes called a "split-level kitchen bar," "stepped kitchen peninsula," or "raised kitchen bar with storage."

---

### Measurements

- **Step height:** 23.5 inches (living room floor to kitchen floor)
- **Railing span to replace:** ~84 inches (7 feet)
- **Desired bar depth:** At least 25 inches

- Metal decorative railing with scrollwork separates the kitchen from the living room
- There is a support column (believed to be removable/non-structural, but needs verification)

- The kitchen is on the right when facing the railing

## Design Configuration (Baseline)

### Continuous Cabinet Structure

A single continuous cabinet extends from bar top down to living room floor, maximizing storage in the step area. Seating on kitchen side with knee clearance overhang.

**Key features:**

- Cabinet sits on kitchen floor AND extends down into step area
- Upper section (kitchen floor to bar top): kitchen-accessible storage
- Lower section (LR floor to kitchen floor): living room-accessible storage
- No toe kick or visual break - seamless cabinet face
- Bar top overhangs into kitchen for seating

**Configuration options:**

#### Flush (Default)
- Single-level counter aligned with step edge
- Simplest build, standard construction
- Counter height (40" from kitchen floor)

#### Tiered
- Counter + raised bar section (+6") facing living room
- True bar height, hides kitchen mess
- More complex build

**Pros:**

- Maximizes storage (uses step area)
- Standard construction techniques
- Structural support for heavy countertop
- Simple, clean design

**Cons:**

- Bar height limited without tiered option
- Custom cabinet depth to fit step area

## Key Measurements & Ergonomics

### Comfortable Bar Dimensions

- **Bar height:** 40-42 inches (from floor where you're sitting)
- **Counter height:** 36 inches
- **Stool height for bar:** 28-30 inches
- **Stool height for counter:** 24-26 inches
- **Seat-to-bar-top distance:** 10-14 inches (ideal)
- **Knee clearance depth:** 12+ inches (minimum for comfort)
- **Seating width:** 24 inches per person

### Height Calculations from Living Room

With 23.5" step + bar elements, the bar top appears quite tall from living room:

- 23.5" step + 36" counter = 59.5" from living room floor
- 23.5" step + 42" bar = 65.5" from living room floor

This means from the living room side, it functions more like a half-wall than a bar.

### Butcher Block Considerations

- Minimum 1.5" thickness recommended for 84" span
- Support brackets every 24-36 inches
- Allow 1/8" gap at walls for wood expansion
- Seal with food-safe finish (oil or polyurethane)

### Component Architecture

Each file follows the same pattern:

```jsx
import React, { useState, useId } from 'react';

export default function ComponentName() {
  // Configuration state with all adjustable values
  const [config, setConfig] = useState({
    stepHeight: 23.5,
    barTopHeight: 40,
    // ... other values
  });
  
  // Unique ID for SVG patterns (prevents conflicts)
  const uniqueId = useId();
  
  // Helper to update config
  const update = (key, value) => setConfig(prev => ({ ...prev, [key]: value }));
  
  // Calculated/derived values
  const calculatedValue = config.something + config.somethingElse;
  
  return (
    <div>
      {/* Inline <style> tag with CSS */}
      <style>{`...`}</style>
      
      {/* Control panel with sliders */}
      <input type="range" onChange={e => update('key', parseFloat(e.target.value))} />
      
      {/* SVG diagrams that react to config state */}
      <svg>
        {/* Drawing logic uses config values */}
      </svg>
    </div>
  );
}
```

### Key Variables to Modify

In the visualization file, the `useState` initialization object contains all adjustable values:

```jsx
const [config, setConfig] = useState({
  stepHeight: 23.5,        // Living room to kitchen floor (fixed)
  barTopHeight: 40,        // Bar height from kitchen floor
  barDepth: 28,            // Total front-to-back depth
  cabinetDepth: 12,        // Depth of cabinet section
  barLength: 84,           // Total length of bar
  stoolHeight: 30,         // Stool seat height
  barTopThickness: 1.5,    // Butcher block thickness
  // Design options
  showPerson: true,
  cabinetStyle: 'mixed',   // 'doors', 'open', 'mixed'
  waterfallEdge: false,
  showLighting: true,
  tieredBar: false,        // Enable raised bar section
  raisedBarHeight: 6,      // Height of raised section
});
```

### Making Modifications

**To change default values:**
Edit the `useState` initialization object.

**To add new options:**

1. Add to `useState` object
2. Add UI control (slider, checkbox, button)
3. Use the value in SVG drawing logic

**To modify the diagrams:**
SVG drawing logic is in arrow functions like `SideSection()`, `LivingRoomElevation()`, etc. These use `config` values to calculate pixel positions.

**Common scale factor:**
Most diagrams use `scale = 1.6` to `2.0` to convert inches to pixels. Example:

```jsx
const scale = 1.8;
const cabinetHeightPx = config.cabinetHeight * scale;
```

### SVG ID Uniqueness

Each SVG uses `useId()` for pattern/gradient IDs to prevent conflicts:

```jsx
const uniqueId = useId();
// Then in SVG:
<pattern id={`${uniqueId}-grid`}>
<rect fill={`url(#${uniqueId}-grid)`}>
```
