# Kitchen Bar Project Brief

## Project Overview

I want to replace an existing metal railing in my split-level home with a bar that includes cabinet storage. The kitchen sits higher than the adjacent living room, creating a unique opportunity for a multi-functional bar/counter space.

This is sometimes called a "split-level kitchen bar," "stepped kitchen peninsula," or "raised kitchen bar with storage."

- **Railing span to replace:** ~84 inches (7 feet)
- **Desired bar depth:** At least 25 inches

## Design Configurations Explored

### Option A: Living-Room-Side Cabinets (Original Concept)
Cabinets mount on the living room side of the step, accessible from the living room. Bar top extends over with knee clearance on the kitchen side for seating.

**Key features:**
- Cabinets face INTO the living room (storage accessible from living room)
- Bar height is independently adjustable (not tied to cabinet height)
- Requires custom framing/construction hanging off the step
- Steel L-brackets needed to support knee-space overhang
- Cabinet height can be very tall (60+ inches from living room floor)
- "Cabinet setback" option tucks cabinets under the bar top, creating an overhang on the living room side

**Pros:**
- Maximum flexibility for bar height
- Living room storage/display
- Can create a floating bar top effect

**Cons:**
- More complex construction
- Requires structural support brackets
- Custom build (not standard cabinets)

---

### Option B: Kitchen-Floor Cabinets
Standard base cabinets sit on the kitchen floor, with bar top extending toward living room and overhanging into kitchen for seating.

**Sub-options explored:**

#### B1: Flush Peninsula
- Cabinets align with step edge
- Bar overhangs into kitchen only
- Living room sees finished back panel
- Simplest construction

#### B2: Setback Peninsula  
- Cabinets pulled back from step edge
- Step remains visible as natural boundary
- Creates landing zone at step level

#### B3: Tiered/Two-Level Top
- Lower counter section (~36") for prep
- Raised bar section (~42") toward living room for seating
- Solves the "counter height isn't bar height" problem
- Raised section hides kitchen mess from bar sitters

#### B4: Ledge + Cabinets (Hybrid)
- Kitchen-floor cabinets with kitchen-side seating
- PLUS separate shelf/ledge mounted on living room side of step
- Ledge at standing height (~42" from LR floor) for drinks, leaning
- Two separate elements, can be phased

#### B5: Upper + Lower (Hybrid)
- Kitchen-floor base cabinets
- PLUS floating display shelves on living room side
- Kitchen storage AND living room display (books, plants, decor)

**Pros of kitchen-floor approach:**
- Standard base cabinet construction
- Easier to install
- More structural support for heavy countertop
- Simpler overall

**Cons:**
- Bar height limited to cabinet height + top (~36" standard)
- Need raiser or tiered top to reach bar height (40-42")
- Less/no functionality from living room side (unless hybrid approach)

## Key Measurements & Ergonomics

### Comfortable Bar Dimensions
- **Bar height:** 40-42 inches (from floor where you're sitting)
- **Counter height:** 36 inches
- **Stool height for bar:** 28-30 inches
- **Stool height for counter:** 24-26 inches
- **Seat-to-bar-top distance:** 10-14 inches (ideal)
- **Knee clearance depth:** 12+ inches (minimum for comfort)
- **Seating width:** 24 inches per person

### Height Calculations from Living Room
With 23.5" step + bar elements, the bar top appears quite tall from living room:
- 23.5" step + 36" counter = 59.5" from living room floor
- 23.5" step + 42" bar = 65.5" from living room floor

This means from the living room side, it functions more like a half-wall than a bar.

### Butcher Block Considerations
- Minimum 1.5" thickness recommended for 84" span
- Support brackets every 24-36 inches
- Allow 1/8" gap at walls for wood expansion
- Seal with food-safe finish (oil or polyurethane)

### Component Architecture

Each file follows the same pattern:

```jsx
import React, { useState, useId } from 'react';

export default function ComponentName() {
  // Configuration state with all adjustable values
  const [config, setConfig] = useState({
    stepHeight: 23.5,
    barTopHeight: 40,
    // ... other values
  });
  
  // Unique ID for SVG patterns (prevents conflicts)
  const uniqueId = useId();
  
  // Helper to update config
  const update = (key, value) => setConfig(prev => ({ ...prev, [key]: value }));
  
  // Calculated/derived values
  const calculatedValue = config.something + config.somethingElse;
  
  return (
    <div>
      {/* Inline <style> tag with CSS */}
      <style>{`...`}</style>
      
      {/* Control panel with sliders */}
      <input type="range" onChange={e => update('key', parseFloat(e.target.value))} />
      
      {/* SVG diagrams that react to config state */}
      <svg>
        {/* Drawing logic uses config values */}
      </svg>
    </div>
  );
}
```

### Key Variables to Modify

In any file, the `useState` initialization object contains all adjustable values:

```jsx
const [config, setConfig] = useState({
  stepHeight: 23.5,        // Living room to kitchen floor
  barTopHeight: 40,        // Bar height from kitchen floor
  barDepth: 28,            // Total front-to-back depth
  cabinetDepth: 12,        // Depth of cabinet section
  cabinetSetback: 3,       // How far cabinet is tucked under bar
  barLength: 84,           // Total length of bar
  stoolHeight: 30,         // Stool seat height
  barTopThickness: 1.5,    // Butcher block thickness
  kitchenOverhang: 12,     // Knee clearance overhang
  // ... design options (booleans, strings)
  showPerson: true,
  cabinetStyle: 'mixed',   // 'doors', 'open', 'mixed'
  waterfallEdge: false,
  showLighting: true,
});
```

### Making Modifications

**To change default values:**
Edit the `useState` initialization object.

**To add new options:**
1. Add to `useState` object
2. Add UI control (slider, checkbox, button)
3. Use the value in SVG drawing logic

**To modify the diagrams:**
SVG drawing logic is in arrow functions like `SideSection()`, `LivingRoomElevation()`, etc. These use `config` values to calculate pixel positions.

**Common scale factor:**
Most diagrams use `scale = 1.6` to `2.0` to convert inches to pixels. Example:
```jsx
const scale = 1.8;
const cabinetHeightPx = config.cabinetHeight * scale;
```

### SVG ID Uniqueness

Each SVG uses `useId()` for pattern/gradient IDs to prevent conflicts:
```jsx
const uniqueId = useId();
// Then in SVG:
<pattern id={`${uniqueId}-grid`}>
<rect fill={`url(#${uniqueId}-grid)`}>
```

When making adjustments, key variables to play with:

1. **Bar height** — Affects stool selection and comfort. 36" = counter stools, 42" = bar stools

3. **Overhang** — Need 12"+ for comfortable knee clearance
4. **Cabinet setback** — 0" = flush, 3-6" = tucked under for visual interest
